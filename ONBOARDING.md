# 负载曲线生成器 · 上手指南（ONBOARDING）

> 给第一次接触这个项目、没有前期上下文的同事。读完这一份 + 扫一遍 `index.html` 就能安全地改。

## 这是什么

`index.html` 是一个**单文件、零依赖**的交互式工具（AlphaESS「Alpha Proposal」光储提案工具的 **Power Profile 模块**）。目标：在缅甸这类拿不到真实区间电表数据的市场，用约 5 个核心输入（月账单电量、柴油、峰值、工作时间、行业类型）**仿真出全年逐时负载曲线**（8760 / 闰年 8784 点），作为下游 PV+储能+柴发微网仿真、ROI 测算的地基。

**技术上没有任何构建步骤**：浏览器直接打开 `index.html` 即可运行，所有 JS/CSS/数据都内联在这一个文件里。

## 核心模型（改代码前必须理解的 8 件事）

**1. 逐时功率主公式（SPEC §4.1 + §5.2）**
```
P(d,h) = P_peak · ( f_base[h] + A_work(m)·w_light·f_prod[h] ) · (1 + ε(d,h))  → 再做逐月电量归一化
```
`w_light`（周内淡日因子）默认 = 1；仅**酒店 / 商场 / 超市**在**周一、周二**取 `0.8`，且**只乘生产部分 `f_prod`**（基载不动）。月电量硬锚不变——淡日的量级差异由后续逐月归一化吸收。

**2. 形状 × 标定解耦**：曲线「长什么样」（形态模板）与「多大量级」（账单标定）分开处理。

**3. 两层基载**：非工作时段 = `B_off%`，工作时段地板 = `B_work%`，生产峰在地板之上由高斯 `g[h]` 升起。都是 `P_peak` 的固定比例。

**4. 电量是硬锚、峰值是软锚**：逐月电量必须等于「账单 + 柴油」（第一优先）；`P_peak` 只是标定目标，**允许小时值突破 P_peak，不设硬帽、不告警**（SPEC 决议 Q9 / P8）。

**5. 生产系数 A_work 逐月由账单反解（SPEC §6 第④步）**：
```
A_work(m) = (E_m − E_base_m) / E_prod_raw_m
```
若某月 `E_m ≤ E_base_m`（基载电量已超过当月总量）→ **退化处理（§4.2）**：`A_work=0`，当月整条曲线等比压低到刚好等于 `E_m`，并告警。

**6. ε 纹理来自 `ε_噪声表.xlsx`（确定性、可复现）**
```
ε(d,h) = 波动幅度[类型] × ( 日因子[日序 d] + 0.4 × 时纹理[星期(d)][h] )
```
- **日因子**（365 天，零均值 σ≈1）：每天一个量级偏差，是「每天不雷同」的主来源，逐日不同。
- **时纹理**（7 星期 × 24 时）：日内质感，按当天星期取列。
- **波动幅度**：每个负载类型一个强度系数（连续/恒载最小如冷库 0.03，充电站最大 0.15）——想更平或更跳只调这一个数。
- ε 之后还有一次**逐月电量归一化**（电量第一优先），所以最终乘子 = `(1+ε) × 当月归一化因子`。
- 这是确定性的：同输入同输出，**不要改成运行时随机**。

**7. 峰位随窗口自适应映射（SPEC §4.3 第 0 步，v1.3 新增）**
`PRESETS` 里的 `seg` 是**模板默认窗口**、`p`（μ/σ）是基于该默认窗口的值。当用户把实际工作窗口改成与默认不同时，产量峰做**仿射映射**（`mapPeaks()`），午休时段同式映射（`mapLunch()`）：
```
μ' = t_on + (μ − t_on^tpl)·W/W_tpl        σ' = σ·W/W_tpl        （W = 窗口时长）
```
- 实际窗口 = 默认窗口 → 恒等，存量结果零变化。
- 修复的缺陷：窗口拉长（如 08–18→08–22）后晚间不再退化为纯 B_work 地板、A_work 不再被顶到 >1。
- 侧栏 μ/σ 输入框始终显示**模板值**（不随窗口变），映射在内部完成。

**8. 日历分两类（SPEC §5.2，v1.3 新增）**
- **普通类目**：周末 + 均匀占位的节假日 → 全天 `B_off`。
- **连续营业类目**（`CONTINUOUS` 集合：商场/超市/餐饮/酒店/充电站/医院/仓储/冷库/**水泥/化工**）：**全周工作**，周末与节假日设置**隐藏且不生效**；唯一例外是**泼水节 Thingyan**（`THINGYAN`，硬编码 4/13–4/16 占位）强制全天 `B_off`。

## 代码结构导航（`index.html` 内 `<script>`）

| 符号 / 函数 | 作用 | 对应来源 |
|---|---|---|
| `const DAY_FACTOR` | 365 天日因子 | ε_噪声表.xlsx「日因子」页 |
| `const HOUR_TEX` | 时纹理 `[hour][周一..周日]` | ε_噪声表.xlsx「时纹理」页 |
| `const PRESETS` | 23 个行业细类形态库，每类含 `amp`（波动幅度） | SPEC §5 + xlsx「波动幅度」页 |
| `readParams()` | 读左侧输入，解析账单（1 或 12 值）、工作段、峰值、amp | SPEC §3 |
| `buildTemplate(p)` | 建单日模板 `base[24]`/`prodRaw[24]`（A_work=1），含段边沿爬坡 | SPEC §4.3/§4.4 |
| `buildCalendar(p)` | 建项目当年日历：每天 `{月, 星期, 日序 doy, 是否工作日}`；周末+节假日=停工；闰年判断 | SPEC §6 |
| `compute(p)` | **主流水线**：逐日/逐月能量 → A_work 与退化 → `epsMult()` 逐日施加 ε → 逐月归一化 → 统计 max/逐月电量，并收集全年逐时序列 `series` | SPEC §6 |
| `epsMult(doy,dow,h)` | `1 + amp·(DAY_FACTOR[doy]+0.4·HOUR_TEX[h][dow])` | ε 公式 |
| `renderMetrics / drawDayChart / drawMonth` | 渲染指标卡、日内 / 全年曲线、逐月电量柱 | SPEC §8/§9 |
| `exportCSV()` | 导出全年逐时数据为 CSV（见下方「导出」） | — |
| `applyPreset / init` | 选类目自动填参、绑定输入事件触发重算、绑定导出按钮 | — |

数据流：任何输入变化 → `update()` → `compute()` → 渲染函数。

## 常见改动怎么做

### 加 / 改一个行业类型（最常见）
在 `PRESETS` 里对应大类下加一行，对照 SPEC §5 形态表 + xlsx 波动幅度表：
```js
"Garment 制衣": {boff:4, bwork:45, seg:"8-18", ld:0.4, ls:"12-13",
                 p:[[10.5,1.5],[15,1.5]], amp:0.1},
//  boff/bwork=两层基载%  seg=模板默认窗口(可多段"6-10,14-18"；也是峰位映射基准)
//  ld/ls=午休预设 δ/时段(v1.3 起客户端仅"有/无"开关、不可改；ls=""则无午休、隐藏该开关)
//  p=[[μ,σ],...]高斯峰(近平直=单个大σ如[[14,8]])  amp=ε波动幅度(取自xlsx)
```
> **连续营业类目**（商场/超市/餐饮/酒店/充电站/医院/仓储/冷库/水泥/化工）自动全周工作、隐藏周末与节假日设置，仅泼水节强制停业；**酒店/商场/超市**再自动套用周一、周二淡日 `w_light=0.8`。这些是按类目写死的业务规则（SPEC §5.2）。

### 调工作日曲线形态
改该类型的 `p`（峰位 μ / 宽度 σ）、`seg`（工作窗口）、`ld/ls`（午休）。尖峰型 σ 取 1.5~2.5，近平直型 σ 取 8~10。

### 更新 ε 纹理数据（改了 `ε_噪声表.xlsx` 之后）
用仓库里的解析脚本重新生成 JS 数组，再贴回 `index.html`：
```bash
pip install openpyxl                      # 首次；或加 --break-system-packages
python3 tools/gen_texture_js.py           # 打印 const DAY_FACTOR / const HOUR_TEX
# 把这两行覆盖 index.html 里对应的两行；
# 若波动幅度也变了，按 stderr 打印的对照表更新各 PRESETS 的 amp
```
> `tools/gen_texture_js.py` 是纹理数据的唯一来源转换器，**不要手改单个数字**——日因子/时纹理是成套、零均值生成的。

### 校验规则（SPEC §8，v1.3 口径）
- **生成前预检**：V9a（某月电量超「P_peak × 工作时段满载」上限）/ V9b（某月负载率 < 10%，峰值虚高）——**只提示、不拦截**，导出时合并成一次弹窗。
- **V11**：工作窗口跨午夜（`t_off ≤ t_on`）→ 阻断，必须改输入。
- **V2**：某月基载电量 ≥ 当月总电量 → 整月等比压低、`A_work=0`、告警。
- 旧 **V1a**（LF≥1 阻断）**已删除**；年负载率 LF 仅作展示（<0.15 提示 V1b）。

## 导出全年逐时数据

右上角 **「⬇ 导出全年 CSV」** 按钮，用当前参数导出模拟出的整年逐时曲线（含 ε、逐月归一化后的真实值）。格式：
```
日期,时刻,星期,类型,负载_kW
2026-01-01,00:00,周四,工作日,32.15
...
```
平年 8760 行、闰年 8784 行；文件名带行业与年份；带 UTF-8 BOM，Excel 打开中文不乱码。导出的数值即三张图背后那条曲线。
